> **项目定位**
> 这是一个每周更新的 GitHub 创意实验室。它不追求一次做很大，而是每周做一个“小而完整、有审美、有切入口”的项目单元：skill、工作流、脚本、agent playbook、前端 demo 或 README-driven product 都可以。

## 🗺️ Lab 规则

| 原则 | 做法 |
| --- | --- |
| 热点要真实 | 优先看近期 GitHub repo 数据、README、stars、更新时间和相邻生态。 |
| 入口要小 | 不和巨头正面对撞，找“垂直人群 + 明确痛点 + 低实现成本”。 |
| 审美要有记忆点 | README 先像产品门面，再像说明书。 |
| 每周只做一个 | 控制体量，保证可以被安装、试用、继续迭代。 |
| 不写流水账 | 更新记录只放文末附录，顶部只留项目介绍和使用方式。 |

## 🧪 如何使用这个仓库

```bash
# 查看本周作品
open weeks/2026-07-06-approval-atlas/README.md

```
